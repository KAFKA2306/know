# 品質維持ワークフロー

リポジトリを「常に最良の状態」に保つためのメンテナンス手順です。

## フロー

### 1. リンク切れ・ビルド検証
`mkdocs.yml` の `validation` 設定に基づき、リンク切れや設定ミスを検出します。

```bash
# ビルド実行（警告・エラーを確認）
uv run mkdocs build
```

**対応:**
- `WARNING` が出た場合は必ず修正する。
- リンク切れ（404）は正しいパスに修正するか、削除する。

### 2. コンテンツの鮮度確認
古い情報が残っていないか確認します。特に年号に注意します。

```bash
# 例: 2025年の記述を探す（現在は2026年）
grep -r "2025" docs/
```

**対応:**
- 情報が古い場合は `update-content` フローに従って更新する。
- 過去の記録として正しい場合はそのままにする。

### 3. フォーマット整形
Markdownのフォーマットを統一します。

这里は将来的に `mdformat` などの導入を検討しますが、現在は手動で「執筆ルール」に従っているか確認します。
- 箇条書きが正しくネストされているか
- コードブロックに言語指定があるか

## 実行頻度
- 週次 または 大きな変更の前後
