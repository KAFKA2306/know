# コンテンツ更新ワークフロー

既存の記事情報が古くなった場合や、定期的な見直しの際に使用します。

## フロー

### 1. ファクトチェック (Fact Check)
最新情報を確認します。

- **Web検索**: 公式サイト、プレスリリースを確認。
- **日付確認**: 「2026年現在」の情報であることを確認。

### 2. コンテンツ更新 (Update)
「正しく、シンプルに」書き直します。

- **Destructive Rewrite**: 既存の文章を無理に維持せず、ゼロベースでシンプルに書き直すことを推奨します。
- **冗長排除**: 不要な前置きや、今は関係ない過去の経緯を削除します。

### 3. 検証 (Verify)
更新した内容が正しいか確認します。
リンク切れがないかチェックします。

```bash
uv run mkdocs build
```

### 4. デプロイ (Deploy)
変更をコミットしてプッシュします。

```bash
git add <file>
git commit -m "update: <topic> for 2026"
git push
```
