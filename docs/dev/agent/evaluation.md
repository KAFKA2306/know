# LLM Evaluation

「良くなった気がする」を殺す層

## 本質

**感覚を排除する装置**。

人は言う：「前より良くなった気がする」

Evaluationは返す：「**数値を出せ**」

## 要素例

| ツール | 特徴 |
|--------|------|
| RAGAS | RAG専用評価。正確性・網羅性・一貫性を自動算出 |
| Galileo | 出力の信頼性・妥当性を評価。継続改善向け |

## よくある失敗

- 人手評価だけに頼る
- 評価指標が曖昧
- 本番後に入れようとする

→ 改善したつもりで劣化していく

## 設計原則

1. **数値がない改善は改善ではない**
2. PoC後半〜本番で必須
3. Observability と必ずセット
